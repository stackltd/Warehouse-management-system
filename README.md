# Warehouse Management System 📦⚙️


Специализированная веб-система учета склада запчастей, разработанная для сервисных центров. 
Проект автоматизирует внутреннюю логистику: отслеживание остатков деталей


## ✨ Основной функционал

* **Учет запчастей:** Мониторинг наличия, добавление, списание и просмотр актуальных остатков на складе.
* **Контроль пользователей:** Логирование всех действий
* **Безопасность:** Полноценная система авторизации пользователей.
* **Смена пароля:** Возможность безопасного обновления учетных данных из личного кабинета.
* **Кэширование сессий:** Использование быстрых хранилищ Redis для менеджмента активных сессий и оптимизации нагрузки.


## 🛠️ Быстрый запуск

### 1. Подготовка окружения
Клонируйте репозиторий на локальный компьютер и перейдите в его корень:
```bash
git clone [https://github.com/stackltd/Warehouse-management-system.git](https://github.com/stackltd/Warehouse-management-system.git)
cd Warehouse-management-system

```

Установите необходимые зависимости проекта:

```bash
pip install -r requirements.txt

```

### 2. Запуск инфраструктуры (Redis)

Для корректной работы авторизации и кэширования запустите контейнер Redis через Docker:

```bash
docker run --name redis-warehouse -p 127.0.0.1:6379:6379 -d redis

```

### 3. Инициализация базы данных (Первый запуск)

Перед тем как запускать само приложение, необходимо подготовить базу данных:
создать таблицы пользователей, структуру склада, а также сгенерировать дефолтного администратора.

Запустите скрипт инициализации из webapp:

```bash
python init_db.py

```

### 4. Запуск веб-приложения

После успешной настройки базы данных и Redis запустите основной сервер приложения:

```bash
python -m webapp.app

```

По умолчанию сервер станет доступен по адресу `http://127.0.0.1:8080` (или иному порту, указанному в настройках вашего Flask-приложения).

## 🔒 Авторизация и Смена пароля

1. **Вход в систему:** Доступ к складскому функционалу ограничен. Для выполнения запросов необходимо пройти аутентификацию на эндпоинте `/login`. 
В ответ система выдает токен/сессию, данные которой валидируются и временно кэшируются в **Redis** для высокой скорости обработки последующих запросов.
2. **Смена пароля:** Пользователь может изменить свой текущий пароль через специализированный защищенный эндпоинт.
Система проверяет валидность старого пароля, хеширует новый с помощью безопасных алгоритмов (например, `bcrypt`) 
и обновляет запись в базе данных, сбрасывая при этом активные сессии в Redis для предотвращения несанкционированного доступа.

## 📡 Описание эндпоинтов (API)

Ниже представлена структура основных эндпоинтов системы управления складом:

### 🔑 Аутентификация и Управление пользователями

* `POST /login` — Аутентификация пользователя (вход в систему), генерация сессии/токена.
* `POST /logout` — Завершение сессии, удаление токена из Redis.
* `PUT /change-password` — Безопасная смена текущего пароля пользователя (требует подтверждения старого пароля).

### ⚙️ Управление складом запчастей. Функционал позволяет

1. Выбрать категорию из списка
2. Выбрать всё
3. Выбрать только активные заказы
4. Установить позицию в один из статусов "к заказу", "заказано", "принято"    
5. Создать новую позицию, выбрав категорию из списка слева и нажав добавить. После этого заполните все нужные поля
6. Редактирование позиции. Измените доступные поля. Если поля недоступны, нужно войти в режим изменения,
нажав на номер позиции. Она станет светиться зеленым. Повторное нажатие выводит из режима редактирования
7. Просмотр финансовой части через кнопку финансы.
8. Сортировка по параметрам осуществляется нажатием названия поля
